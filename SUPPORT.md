# Support

Use GitHub issues for project support.

## Where to Ask

| Topic | Place |
| --- | --- |
| CLI behavior, command output, or local state | GitHub issue |
| Documentation errors in README or bookshelf | GitHub issue |
| Security concern | See [SECURITY.md](SECURITY.md) |
| Contribution process | See [CONTRIBUTING.md](CONTRIBUTING.md) |

## Before Opening an Issue

Check the command help first:

```bash
seekmoon --help
seekmoon <command> --help
```

For output contracts, inspect:

```bash
seekmoon <command> --shape
seekmoon <command> --schema
```

For documentation, prefer the maintained bookshelf:

- [Package Reuse Ecosystem](https://michengliang.github.io/seekmoon/books/08-package-reuse-ecosystem/book.html)
- [SeekMoon: MoonBit Package Discovery Workbench](https://michengliang.github.io/seekmoon/books/09-seekmoon-cli-discovery-workbench/book.html)

`docs/` contains research notes, raw investigation material, and validation
reports. It is not the primary support surface.
