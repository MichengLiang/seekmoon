== WP03：Platform Runtime and Storage

=== 工作包身份

WP03 实现 SeekMoon 的 host capability boundary、runtime composition 和 storage boundary。它把 clock、filesystem、HTTP client、command runner、path resolution、stores、source registry、service registry 和 renderer registry 装配为一次进程执行可消费的 Runtime。

=== 实现边界

WP03 owns path construction and host capability abstraction. Services decide when to create snapshots、records、reports、probe directories or source directories. Stores do not decide adoption conclusions, source priority or output shape.

Project storage lives under `.seekmoon/` within the current project context. Reusable cache lives under `$XDG_CACHE_HOME/seekmoon/`. Probe and source resolution paths are controlled side effects. Current project manifest mutation requires explicit command surface and cannot be hidden inside store methods.

=== 测试责任

WP03 tests use `t.TempDir()` and fake platform implementations. Tests must cover path construction, XDG fallback, project storage directories, cache storage directories, atomic write behavior where required, session candidate mapping, record/report path sanitization and command log path generation.

=== 审核尺子

审核者检查四项。

* Runtime construction order follows process environment -> path resolution -> platform clients -> source readers -> stores -> services -> renderer -> runtime value -> CLI command tree.
* `context.Context` can reach HTTP requests, repository enrichment, Moon CLI commands, downloads and probe commands.
* Store package owns mechanics, not business decisions.
* Side effect paths match 9 号书 storage contract.
