== WP07：Repository and Skill Sources

=== 工作包身份

WP07 实现 repository signal reader、Skills API reader、skill asset reader and runwasm coordinate derivation。它处理 maintenance evidence and executable Wasm entries，不处理 library dependency adoption。

=== 实现边界

Repository signal answers maintenance questions. It does not answer registry publication facts. Repository main branch may differ from registry published source. Repository enrichment only appears when requested by command or report evidence refs. Repository enrichment failure changes repository signal state and must not fail unrelated manifest or module profile fields.

Skill entry belongs to executable path. It is not a library dependency candidate. Skill checksum verifies artifact content, not publishing identity、build provenance or vulnerability status. Skill record and report can exist, but library adoption conclusion remains separate.

=== 测试责任

WP07 tests cover:

* Repository URL parsing and unsupported repository state。
* GitHub archived、pushed_at、license、release、issue、PR、workflow、README/tests/examples mapping。
* Repository unreachable maps to `failed` repository signal without failing module profile。
* Skills API list and detail parsing。
* Empty skill package maps to root marker。
* Skill asset fetch maps SKILL.md、wasm and checksum states。
* Pinned runwasm coordinate derivation。

=== 审核尺子

审核者检查五项。

* go-github and OAuth2 imports stay inside `internal/source`。
* Repository signal never replaces published source resolution。
* SkillEntry and ModuleSummary remain different candidate models。
* Checksum wording and state never imply provenance。
* Unrequested enrichment does not enter output objects。
