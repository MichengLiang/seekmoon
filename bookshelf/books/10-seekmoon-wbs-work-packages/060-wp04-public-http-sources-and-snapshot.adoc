== WP04：Public HTTP Sources and Snapshot

=== 工作包身份

WP04 实现 Mooncakes public HTTP sources 和 snapshot creation 所需 source readers。它读取 Modules API、Statistics API 和 Manifest API，把 HTTP result 转换为带 source label、fetch time、status、parse state、raw reference 和 normalized value 的 source result。

=== 实现边界

Modules API 是 library module search 的基础 input surface。它不提供 downloads、build status、package tree、API details、target proof、source files、repository activity 或 vulnerability status。WP04 不得在 modules reader 中派生质量判断。

Statistics API 给出 registry snapshot counters。Counters 只说明调查时的数据口径，不参与单个候选质量判断。

Manifest API 是单 module profile 的 registry profile 来源。Manifest metadata 是开放对象；WP04 只规范化已知字段并保留 raw metadata。Target metadata 只能作为 source evidence，不能被解释为 probe proof。

=== 测试责任

WP04 tests cover:

* HTTP 2xx with valid JSON maps to `present` source action。
* HTTP timeout or non-2xx maps to `failed` source action with URL and error。
* JSON parse failure records parse state。
* Empty `description`、`keywords`、`repository`、`license` become `missing` evidence fields。
* Manifest `module` and `name` mismatch enters failure path for profile construction。
* `sync` records partial source failure in snapshot state。

=== 审核尺子

审核者检查五项。

* Source labels are stable: `modules_api`、`statistics_api`、`manifest_api`。
* Fetch result carries URL、fetched_at、status、parse_state、raw_ref and error when present。
* Modules API reader does not fetch asset or repository data。
* Manifest reader preserves raw metadata and normalized projection。
* Snapshot failure semantics do not erase partial successful sources。
