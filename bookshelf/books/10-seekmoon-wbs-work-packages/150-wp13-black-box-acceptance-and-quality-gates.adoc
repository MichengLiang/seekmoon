== WP13：Black-box Acceptance and Quality Gates

=== 工作包身份

WP13 turns public behavior into executable acceptance and engineering quality gates。It owns journey tests、black-box CLI tests、schema validation tests、golden output tests、integration boundaries and `just ci` gate composition。

=== 输入

WP13 receives all public command surfaces, source reader fakes, stores, renderer, schemas, golden fixtures and quality commands from prior work packages。

=== 输出

[cols="1,2", options="header"]
|===
|路径 |对象

|`internal/testutil/fixture.go`
|Reusable source, service and CLI fixtures。

|`internal/testutil/golden.go`
|Golden file helpers and update policy。

|`tests/blackbox/`
|CLI black-box acceptance tests for A1 to A16。

|`tests/journey/`
|Library、skill、pipeline and failure recovery journey tests。

|`tests/integration/`
|Opt-in real network、Moon CLI、GitHub and probe mutation tests。

|`justfile`
|Final `just ci` composition and integration test commands。
|===

=== 实现边界

Default tests are offline, repeatable and free of external side effects. Tests using real network、real Moon CLI、GitHub API、source zip downloads or probe project mutation run only when explicit integration environment variables are present.

Coverage exposes test reach. It is not the sole acceptance condition. A passing quality gate does not cancel black-box behavior checks. Black-box acceptance remains language-independent public behavior evidence.

=== 审核尺子

审核者检查五项。

* A1 to A16 each map to executable test names。
* Default tests do not require network, Moon CLI mutation or GitHub token。
* Integration tests are opt-in and visibly named。
* Golden output changes require intentional review。
* `just ci` composes the same gates as local development and CI policy。
