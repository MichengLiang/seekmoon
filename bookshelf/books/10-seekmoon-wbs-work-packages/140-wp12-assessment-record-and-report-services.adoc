== WP12：Assessment, Record and Report Services

=== 工作包身份

WP12 实现 `probe`、`record`、`report` and `raw` service flows。These services generate local derived evidence, persist adoption decisions, project investigation reports and expose raw payloads for audit。

=== 实现边界

Probe creates local derived evidence. It proves only current environment、current version、current target and current command sequence. Probe failure does not prove upstream metadata is wrong.

Record persists candidate、version、snapshot、evidence refs、conclusion、not confirmed items and note. Conclusion enum is fixed: `adopt`、`adopt-with-adapter`、`continue-verification`、`contribute-upstream`、`fork`、`build-own`、`reject-for-now`.

Report lists only sources actually used by records and evidence refs. If repository enrichment was not executed, report does not invent repository activity. If probe was not executed, report does not claim validation success or failure.

Raw output preserves upstream field names and source status. It bypasses ordinary normalization for audit; it does not become normal discovery path.

=== 测试责任

WP12 tests cover:

* Probe command sequence uses isolated probe path。
* Command results include command、cwd、exit code、status and log path。
* Target build failure produces ProbeResult with failed status and logs。
* Record validates conclusion enum and writes evidence refs。
* Report omits sources not present in evidence refs。
* Raw service returns raw payload envelope with source URL/path、fetch status and timestamp。

=== 审核尺子

审核者检查五项。

* Probe does not mutate upstream facts or adoption record automatically。
* Record does not accept arbitrary conclusion strings。
* Report does not add unobserved sources。
* Raw service does not normalize or hide upstream fields。
* Service flow remains store-backed and renderer-independent。
