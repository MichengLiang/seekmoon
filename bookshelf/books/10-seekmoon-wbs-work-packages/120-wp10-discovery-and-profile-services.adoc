== WP10：Discovery and Profile Services

=== 工作包身份

WP10 实现 `doctor`、`sync`、`search`、`view` 和 `compare` service flows。These services create environment status、snapshot、library module candidates、module profiles and evidence comparison objects。

=== 实现边界

`doctor` reads environment and reports status. It does not create snapshot、update registry or write record.

`sync` creates dated evidence snapshot. It records `moon update` result、Modules API、Statistics API、local registry index and toolchain versions. Candidate ranking and adoption decision remain outside sync.

`search` generates library module candidates from query and snapshot. It may fetch visible-window manifest enrichment for downloads and build status. It does not read package data、published source or probe evidence.

`view` presents one module profile. It reads Manifest API and module index asset. It does not expand all API symbols. Repository activity enters output only when requested enrichment is loaded.

`compare` aligns loaded evidence fields and does not generate quality score.

=== 测试责任

WP10 tests cover:

* Doctor reports toolchain、paths、network and project context using fake capabilities。
* Sync writes snapshot with source status and partial failure state。
* Search uses latest snapshot or transient modules fetch。
* Search rank includes match evidence and writes session candidate map。
* Manifest enrichment failure appears in result field state。
* View reads manifest and module index and reports package count/index state。
* Compare aligns multiple candidates without producing score。

=== 审核尺子

审核者检查五项。

* Discovery services never turn downloads/build/repository presence into quality proof。
* Session-local candidate numbers are bound to snapshot/session boundary。
* Target unknown remains unknown before metadata/probe evidence。
* Compare output is evidence alignment, not ranking or recommendation。
* Services compose source results and stores; they do not render directly。
