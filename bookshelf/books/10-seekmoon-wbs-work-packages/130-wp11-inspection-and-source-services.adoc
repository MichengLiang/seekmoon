=== 实现边界

`api` resolves module version, loads module index, derives package relpath and fetches package data. Package path failure is resolved against module index and error surface lists known package paths. `resource.json` unavailable does not block package data.

`source` records every source resolution attempt: `moon fetch`、source zip、local cache、core local source and repository fallback signal. `selected_source` is derived from successful attempts. Repository fallback must be marked as repository signal, not registry published source.

`skill search` and `skill view` operate on executable entries. Skill execution verification and library dependency probe remain separate paths.

=== 测试责任

WP11 tests cover:

* API service loads module index before package data。
* Package relpath is derived from canonical package path。
* Package path absent returns error surface data with known package list。
* Source resolution records all attempts and chooses selected source deterministically。
* `moon fetch` failure falls through to other attempts when contract allows。
* Skill search uses Skills API and not Modules API。
* Skill view loads SKILL.md、wasm/checksum asset states and derives runwasm coordinate。

=== 审核尺子

审核者检查五项。

* `api` does not re-run search; it consumes resolved module/package input。
* Source resolution does not collapse failed/unavailable/unknown attempts。
* Repository fallback wording does not claim published source。
* Skill profile does not enter library adoption conclusion。
* Services preserve raw source metadata for JSON/raw audit surfaces。
