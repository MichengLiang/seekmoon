=== 实现边界

`module_index.json` 当前事实表面使用 `childs`。Parser may tolerate `children`, but canonical authored surface remains `childs`. Package relpath must be derived from package path and module coordinate. Root package relpath is empty.

`package_data.json` preserves raw signature. Plain text signature is derived projection and cannot replace raw signature. Source location missing state remains evidence state, not command failure by itself.

`resource.json` is optional. HTTP 404 maps to `unavailable`, not package absence, API data absence or module absence. Source zip failure does not prove module absence and must remain one source attempt in SourceResolution.

=== 测试责任

WP05 tests cover:

* Module index tree parsing with `childs`。
* Package relpath derivation for root and nested package paths。
* Package path absent error data includes known package paths。
* Package data parses types、traits、errors、typealias、values、misc and methods。
* HTML signature generates plain signature while preserving raw signature。
* `resource.json` 404 maps to `unavailable`。
* Source zip redirect and zip reading produce files summary。

=== 审核尺子

审核者检查五项。

* Asset URL construction uses manifest version and derived package relpath。
* Package relpath derivation is not handwritten per command。
* Resource asset state cannot block package data API profile。
* Raw signature and raw payload audit material remain available for JSON/raw surfaces。
* Source zip is a source attempt, not the only published source route。
